## Project Overview

**AJAZZ Deck** is a Linux daemon that interfaces with the AJAZZ AK820 (AKP153E) macro pad/stream deck device. It loads button configurations from YAML and executes shell commands or scripts when buttons are pressed.

### Key Characteristics
- Uses the **Mirabox StreamDock SDK** (Python-SDK submodule in `sdk/`)
- Runs as a daemon with single-instance enforcement via PID file
- Supports clipboard integration via WSL Windows integration
- Includes HID protocol testing/debugging utilities
- Python 3.12+, using `uv` package manager

## Architecture

### Core Application Flow (`deck.py`)
1. **Initialization**: Logging setup, PID file check (single-instance enforcement)
2. **Config Loading**: Parses `buttons.yaml` (via `AJAZZ_CONFIG` env var or default)
3. **Device Detection**: Finds AJAZZ device via Mirabox SDK (`StreamDock` class)
4. **Button Loop**: Registers callback, listens indefinitely for button presses
5. **Execution**: When button pressed, executes configured command/script with timeout

### Button Configuration Format (`buttons.yaml`)
Buttons can be:
- **Simple**: `button_id: "command string"`
- **Complex**: `button_id: {label, image, type, script/command}`

Supported types:
- `shell` (default): Safe execution via `shlex.split()` (no shell interpretation)
- `clipboard`: Shell execution with pipes (for clipboard copy commands)
- `script`: Shell execution (complex multi-line scripts)

### Key Modules
- **deck.py**: Main daemon (config loading, device management, command execution)
- **raw_hid.py**: HID protocol debugging tool (direct /dev/hidraw communication)
- **main.py**: Stub file (generated by uv scaffold, unused)

## Configuration & Environment

### `buttons.yaml` Structure
```yaml
buttons:
  1:
    label: "Button Name"
    image: "icons/button.png"  # optional, for display
    type: "shell"              # shell (default), clipboard, or script
    command: "simple command"  # for shell type
    script: |                  # for clipboard/script types
      multi-line
      bash script
```

### Environment Variables
- `AJAZZ_CONFIG`: Path to button config (defaults to `buttons.yaml` in project dir)

### Important Paths
- `deck.log`: Rotating file log (max 1MB, 3 backups)
- `deck.pid`: Runtime PID file (auto-cleaned on shutdown)
- `sdk/Python-SDK`: Mirabox StreamDock SDK submodule

## SDK Integration

The Mirabox StreamDock SDK is included as a git submodule in `sdk/`. Key classes:
- `StreamDock.DeviceManager`: Enumerate devices
- `StreamDock.Devices.StreamDock`: Device instance with `set_key_callback()`
- `StreamDock.InputTypes.EventType`: Button/encoder events

The SDK path is added to `sys.path` at startup (see `deck.py:7`).

## Command Execution Safety

**Important Security Notes:**
- **Safe mode** (`shlex.split()`): Used by default for `shell` type — prevents shell injection
- **Shell mode** (`shell=True`): Used for `clipboard` and `script` types — enables pipes and complex logic
- Commands have a **10s timeout** (shell) or **5s timeout** (safe mode)
- Always validate config files and button scripts — malicious YAML can execute arbitrary commands

## Common Development Tasks

### Adding a New Button
1. Edit `buttons.yaml`: Add button entry with ID, label, command/script
2. Test config syntax: `python -c "import yaml; yaml.safe_load(open('buttons.yaml'))"`
3. Restart daemon: Kill old process, run `python deck.py` again

### Modifying Logging
Logging is configured in `setup_logging()` (deck.py:32-48). The file handler uses `RotatingFileHandler` with 1MB file size limit.

### Testing Device Connection
Run `raw_hid.py` to debug HID communication and verify device is responsive.

## Notes & Gotchas

- **Device Detection**: Uses VID=0x0300, PID=0x3010 (Mirabox StreamDock SDK default)
- **Single Instance**: If daemon crashes without cleaning `deck.pid`, remove it manually before restarting
- **WSL Integration**: Button 1 uses `/mnt/c/Windows/System32/clip.exe` for clipboard — WSL-specific
- **SDK Submodule**: The `sdk/` directory is a git submodule; ensure it's cloned with `--recursive`
